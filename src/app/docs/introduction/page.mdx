# Authentication

Cal.com uses <span style={{ color: "#0b79d0", fontWeight: "bold" }}>API keys</span> to authenticate all API requests. This ensures that only authorized clients can access your account and perform operations. API keys can be viewed and managed in your account under the <span style={{ color: "#0b79d0", fontWeight: "bold" }}>Security</span> settings.

---

Cal.com provides **two types of API keys**:

-   **Test Mode Keys**: <span style={{ color: "#f39c12" }}>Used for testing and development.</span>  
    Start with: <span style={{ color: "#27ae60", fontWeight: "bold" }}>cal\_</span>

-   **Live Mode Keys**: <span style={{ color: "#e74c3c" }}>Used for production.</span>  
    Start with: <span style={{ color: "#c0392b", fontWeight: "bold" }}>cal*live*</span>

## üõ†Ô∏è How to Use API Keys

You can authenticate your requests either by **query parameter** or **HTTP header**. Using headers is recommended for security.

### 1. Query Parameter

Append the `apiKey` to your request URL:

```bash
curl "https://api.cal.com/v1/endpoint?apiKey=cal_live_xxxxxx"
```
